Linear Decoder for Known Types

This tutorial shows how to create a linear decoder object when using data types that are known to pressio. Recall that a linear decoder represents a mapping of the form:

where is a matrix (for now assumed constant), is the reduced state (or generalized coordinates), and is the full-order model (FOM) state. A linear decoder is thus fully defined by its Jacobian matrix.

Types Description

  • : this is the FOM state and, therefore, it is a data structure of your application. For example, if your application uses Trilinos/Tpetra, the FOM state can be, e.g., a Tpetra vector. Typically, is a large (distributed) vector.
  • : this is the (dense) matrix of this linear mapping (for ROMs, this can be, e.g., POD modes). This should also be a data structure from your application. For example, if you are using Trilinos/Tpetra, this matrix can be, e.g., a Tpetra MultiVector. Typically, is a large (distributed) dense matrix.
  • : this is the ROM state. ROM data strutures generally involve small, dense operators that fit well on a single node. In pressio, regardless of what the FOM types are, the data types of the ROM operators are pressio wrappers of either Eigen (always enabled) or Kokkos (optional dependency) types. The user can choose which one to use. The reason of this choice is that ROM operators are always small and dense and fit well on a single node, as explained [here]( todo).

For demonstration purposes, here we use Eigen types for the FOM. For other types already known to pressio, this would work similarly.

Main function

int main(int argc, char *argv[])
{
  std::cout << "Running tutorial\n";

  using native_fom_state_t = Eigen::VectorXd;
  using native_phi_t       = Eigen::MatrixXd;
  // wrap native types with pressio containers
  using fom_state_t = pressio::containers::Vector<native_fom_state_t>;
  using phi_t       = pressio::containers::DenseMatrix<native_phi_t>;

  // create native phi, and for simplicity fill with ones
  native_phi_t phiNative(6, 3);
  phiNative.setConstant(1.);

  // construct linear decoder
  using decoder_t = pressio::rom::LinearDecoder<phi_t, fom_state_t>;
  decoder_t decoder(std::move(phiNative));

  // construct the reduced state, fill with 2s
  using rom_state_t = pressio::containers::Vector<Eigen::VectorXd>;
  rom_state_t yRom(3);
  pressio::ops::fill(yRom, 2.);

  // apply mapping: computes yFom = phi yRom
  fom_state_t yFom(6);
  decoder.applyMapping(yRom, yFom);

  // check
  for (auto i=0; i<yFom.extent(0); ++i)
    std::cout << "i= " << i
          << ", yFom(i) = "  << yFom(i)
          << ", expected = " << 6.
          << "\n";

  return 0;
}

Code Breakdown

The main function starts with type aliasing:

using native_fom_state_t = Eigen::VectorXd;
using native_phi_t       = Eigen::MatrixXd;
// wrap native types with pressio containers
using fom_state_t = pressio::containers::Vector<native_fom_state_t>;
using phi_t       = pressio::containers::DenseMatrix<native_phi_t>;

where we explictly create pressio wrapper types for the FOM state and decoder Jacobian . This is needed because pressio operates on these wrapper types.

We then create a native matrix:

// create native phi, and for simplicity fill with ones
native_phi_t phiNative(6, 3);
phiNative.setConstant(1.);

Note that this is a native type, so in this case Eigen.

The decoder object is instantiated here:

using decoder_t = pressio::rom::LinearDecoder<phi_t, fom_state_t>;
decoder_t decoder(std::move(phiNative));

Note that the native object phiNative is moved from (since Eigen supports move semantics) to avoid a deep copy. After the move, phiNative is left in undefined state, so it should not be used. One can also do decoder(phiNative) which implies a deep copy of the matrix, and obviously, if the matrix is large avoiding a copy is useful.

To perform the computation, the LinearDecoder class exposes an applyMapping method:

// apply mapping: computes yFom = phi yRom
fom_state_t yFom(6);
decoder.applyMapping(yRom, yFom);
